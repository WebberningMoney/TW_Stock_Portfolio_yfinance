# 台股庫存、損益與配息管理 v1.5

本版採用：

- **yfinance API**：商品清冊、行情、歷史現金股利與股票分割。
- **Yahoo 台灣股利政策頁爬蟲**：只針對已登錄持股，補入已公告但尚未出現在 yfinance 歷史序列的股利。
- **SQLite**：持股、行情、股利／分割與同步紀錄。
- **Tkinter + Matplotlib**：桌面 GUI 與每月配息圖表。

## v1.5 新增與修正

### 1. LOG 清楚區分資料來源

更新持股股利時，LOG 會使用明確前綴：

```text
[API／歷史] 6284.TWO：第 1/3 次嘗試
[API／歷史] 完成：6284.TWO 15 筆股利／分割

[爬蟲／Yahoo 台灣已公告] 6284.TWO：第 1/3 次嘗試
[爬蟲／Yahoo 台灣已公告] 完成：6284.TWO 8 筆，其中尚未發放 1 筆
```

因此可以直接判斷每筆資料是從 yfinance API 還是 Yahoo 台灣網頁爬蟲取得。

### 2. API 與爬蟲重複資料自動整合

下列欄位完全相同時，認定為同一筆資料：

- 除息／事件日
- Yahoo Symbol
- 股票代號
- 股票名稱
- 數值
- 事件類型（額外安全條件）

程式會保留欄位較完整的一筆。若完整度相同，優先保留 Yahoo 台灣爬蟲資料，因為通常另有：

- 所屬期間
- 現金發放日
- 公告狀態

LOG 範例：

```text
[資料整合] 整合重複資料：0056.TW 2026-07-21 DIVIDEND 1.35；
API／yfinance 歷史＋爬蟲／Yahoo 台灣已公告 共 2 筆 → 保留 爬蟲／Yahoo 台灣已公告
```

整合後，「已載入資料 → 股利／分割」只顯示一筆。

### 3. 避免已公告股利與歷史模式預估重複

舊版只用月份判斷是否已存在資料，可能發生：

```text
6284 佳邦：2026 年 7 月已有公告股利
但又依 2025 年 8 月模式產生 2026 年 8 月預估
```

v1.5 改為將當年度已發生／已公告事件，與最近一年度歷史事件做**一對一配對**。配對時允許發放月份前後移動：

```text
去年 8 月 ↔ 今年公告 7 月
```

會被視為同一次年度股利，不再額外產生 8 月預估。季配或月配商品仍會保留尚未被當年度公告覆蓋的其他期次。

### 4. 圖表與視窗高度增加

- 預設視窗提高到 `1600 × 1040`，並依實際螢幕可用高度縮放、置中。
- 圖表 Figure 高度由 5.0 增加到 5.8 吋。
- 圖表 Canvas 高度由 470 增加到 540。
- 配息分頁預設約 74% 高度給圖表。
- Matplotlib 上方邊界增加，避免圖表標題及圖例被裁切。
- 圖表與下方明細仍可拖曳分隔線調整。

## 保留舊資料庫

資料庫位置：

```text
data/portfolio_yfinance.db
```

使用全新資料夾時，將舊版資料庫複製到新版 `data/` 即可。v1.5 不需要重建資料庫，也沒有新增必須手動處理的欄位。

執行一次「更新持股股利／分割＋已公告股利」後，既有的 API／爬蟲完全重複資料也會自動整理。

## 建議操作順序

```text
① 選擇類型並建立商品清冊
② 更新全部商品行情
輸入並儲存持股
③ 更新持股股利／分割＋已公告股利
```

完成後可在：

- 「每月配息估算」查看已實現、未領及每月組成。
- 「已載入資料 → 股利／分割」查看整合後的股利／分割資料。
- 「下載進度／LOG」查看 API、爬蟲、重試及資料整合結果。

## 資料限制

Yahoo 台灣股利政策頁不是正式公開 API，頁面結構改版、反爬蟲措施、網路問題或商品資料缺漏，都可能導致解析失敗。因此本程式：

- 只抓取使用者已登錄持股，不爬取全市場股利頁。
- 每檔請求之間保留短暫間隔。
- 單檔最多重試三次。
- 保留 yfinance 歷史資料作為備援。
- 在 LOG 明確列出失敗代號與資料來源。

金額依「目前持有股數」估算，不等同券商實際入帳紀錄；正式交易、稅務或法律用途仍應核對投信、公司及交易所公告。
