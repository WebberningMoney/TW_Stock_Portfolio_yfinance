# 台股庫存、損益與配息管理 v1.4

本版採用：

- **yfinance**：商品清冊、行情、歷史現金股利、股票分割。
- **Yahoo 台灣股利政策頁補強器**：只針對已登錄持股，補入已公告但尚未出現在 yfinance 歷史股利序列的資料。
- **SQLite**：持股、行情、股利與同步紀錄。
- **Tkinter + Matplotlib**：桌面 GUI 與每月配息圖表。

## v1.4 新增與修正

### 1. 補強「已公告但尚未領到」的股利

按下：

```text
③ 更新持股股利／分割＋已公告股利
```

程式會依序處理每一檔持股：

1. 用 yfinance 下載歷史股利與股票分割。
2. 讀取 Yahoo 台灣股利政策頁：

```text
https://tw.stock.yahoo.com/quote/{Yahoo Symbol}/dividend
```

3. 補入：
   - 股利所屬期間
   - 每股現金股利
   - 除息日
   - 現金股利發放日
   - 公告／發放狀態
4. 依「代號＋除息日」合併資料，Yahoo 台灣股利政策頁優先於 yfinance，避免重複計算。

例如 0056 使用正確 Yahoo Symbol：

```text
0056.TW
```

不是 `0056.TWO`。

### 2. 已實現與未領判定更精確

- 有現金發放日：發放日已到才列為「已實現」。
- 發放日尚未到：列為「未領／預估」。
- 純 yfinance 歷史資料沒有發放日時：暫以除息日判定。
- 尚未公告的未來月份：仍可用最近歷史發放模式估算。

金額仍使用「目前持有股數」估算，不等同券商實際入帳紀錄。

### 3. 儲存持股後游標回到股票代號

完成儲存與清空欄位後，輸入焦點會回到「股票代號」，方便連續輸入。

### 4. 配息圖表高度修正

- 圖表高度由 3.6 吋提高至 5.0 吋。
- 配息分頁預設將約 68% 高度分配給圖表。
- 圖例較多時自動改成兩欄。
- 圖表與下方明細仍可拖曳分隔線調整。

### 5. 單一項目失敗時重試三次

以下作業會針對失敗商品逐項重試三次：

- 全部商品行情
- 持股行情
- yfinance 歷史股利／股票分割
- Yahoo 台灣股利政策頁
- 清冊中缺失商品的代號解析

LOG 會顯示：

```text
行情 0056.TW 第 1 次失敗：...
行情重試成功：0056.TW
```

或：

```text
最終失敗：XXXX.TW 已公告股利頁；重試 3 次後仍失敗
```

### 6. 「公司行動」文字調整

GUI 與 LOG 改用較清楚的名稱：

```text
歷史股利／分割
股利／分割資料
Yahoo 台灣股利政策
```

## 保留舊資料庫

資料庫位置：

```text
data/portfolio_yfinance.db
```

使用全新資料夾時，先將舊資料庫複製到新版 `data/`。程式啟動時會自動新增：

```text
period
payment_date
announcement_status
```

不需要手動刪除或重建資料庫。

## 建議操作順序

```text
① 選擇類型並建立商品清冊
② 更新全部商品行情
輸入並儲存持股
③ 更新持股股利／分割＋已公告股利
```

完成後可在：

- 「每月配息估算」查看已實現、未領及每月組成。
- 「已載入資料 → 股利／分割」查看除息日、發放日、所屬期間、公告狀態及來源。
- 「下載進度／LOG」查看逐檔處理及重試結果。

## 網頁擷取限制

Yahoo 台灣股利政策頁不是正式公開 API，頁面結構改版、反爬蟲措施、網路問題或商品資料缺漏，都可能導致解析失敗。因此本程式：

- 只抓取使用者已登錄持股，不爬取全市場。
- 每檔請求之間保留短暫間隔。
- 單檔最多重試三次。
- 保留 yfinance 歷史資料作為備援。
- 在 LOG 明確列出失敗代號。

資料應用於個人研究與投資管理；正式交易、稅務或法律用途仍應核對投信、公司及交易所公告。
