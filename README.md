# 台股庫存、損益與配息管理 v2.1

v2.1 延續 v2.0.1 的多來源股利架構，重點改善股利來源選擇、配息頁面空間與抓取參數管理。

## v2.1 新功能

### 1. 股利資料來源可選

「資料來源與同步」區可選：

- **兩者（建議）**：yfinance 歷史股利／分割＋Yahoo 台灣已公告股利。
- **僅 yfinance API**：只更新歷史股利與股票分割。
- **僅 Yahoo 台灣爬蟲**：只更新已公告股利政策，不重新抓歷史 API。

未選取的來源不會刪除資料庫中既有紀錄；同步完成後仍會整理跨來源重複事件。

### 2. 每月配息頁改為左右切版

- 左側：每月堆疊長條圖。
- 右側：12 個月摘要、選定月份組成、全年股利明細。
- 中間分隔線可拖曳。
- 預設圖表約占 58% 寬度，表格約占 42%。

### 3. 視窗使用更多螢幕高度

主視窗預設使用接近完整螢幕高度，並保留少量 macOS 選單列與 Dock 空間。

### 4. 資料同步名稱調整

原本「yfinance 資料同步」改為「資料來源與同步」，更符合目前同時使用：

- yfinance
- Yahoo 台灣股利政策頁爬蟲
- SQLite 本機快取

### 5. 抓取參數／單筆測試分頁

新增可保存的設定：

- 行情批次大小
- 行情期間與間隔
- 下載執行緒數與 yfinance 請求逾時
- 行情批次間隔
- yfinance 價格修復
- 歷史股利／分割期間與單檔間隔
- 單一項目重試次數
- 重試退避時間
- 爬蟲單檔間隔
- 爬蟲逾時時間
- Screener 每頁筆數與最大頁數

設定保存於：

```text
data/app_settings.json
```

另提供「單筆資料測試」，可在大量同步前先測試：

- 行情
- yfinance 歷史股利／分割
- Yahoo 台灣公告股利爬蟲
- 全部項目

單筆測試不會寫入行情或股利資料表。

## 建議參數

一般個人使用建議保留預設：

```text
行情批次檔數：50
下載執行緒：8
yfinance 請求逾時：15 秒
行情批次間隔：0.2 秒
歷史資料單檔間隔：0.2 秒
重試次數：3
重試退避：1 秒
爬蟲單檔間隔：0.5 秒
爬蟲逾時：25 秒
行情期間／間隔：1mo／1d
歷史股利期間：max
```

遇到 Yahoo 限流或連線不穩時：

1. 將行情批次降至 20～30。
2. 將執行緒降至 4。
3. 將爬蟲間隔提高至 1～2 秒。
4. 重試提高至 4～5 次，但不要設定無限重試。
5. 先使用單筆測試確認代表性商品。

## 資料來源與限制

- 歷史行情、歷史股利與股票分割：yfinance。
- 已公告但尚未發放股利：Yahoo 台灣股利政策頁爬蟲。
- Yahoo 台灣網頁結構可能改版。
- yfinance 與爬蟲皆非台灣主管機關正式結算資料。
- 本工具資料僅供個人研究與資產管理，不構成投資建議。
